# Terminal demo (fuse_check)

`fuse-check-demo.gif` is the README's terminal demo: an AI agent proposes an edit to
`OrderService.cs` that misuses an API (`OrderOptions.MaxItemCount`, a member that does not
exist), `fuse_check` returns the real compiler diagnostic and a repair packet before the
edit lands, and the corrected edit checks clean. The recording is reproducible and its
output blocks are the tool's real responses, never hand-written.

## Files

- `fuse-check-demo.mjs` - the driver. Spawns `fuse mcp serve`, calls the real `fuse_check`
  MCP tool against `tests/fixtures/OrderingApp` with the broken edit and then the corrected
  edit, and writes the asciinema cast from the real responses.
- `fuse-check-demo.cast` - the generated asciinema v2 cast (regenerated by the driver).
- `fuse-check-demo.gif` - the rendered GIF embedded in the README.

## Regenerate

Requires Node and a v4 `fuse` (the built binary or the installed global tool).

```bash
# 1. Regenerate the cast from a real run (pass the built binary if the global tool is older).
node assets/demo/fuse-check-demo.mjs --fuse src/Host/Fuse.Cli/bin/Release/net10.0/fuse.exe

# 2. Render the cast to the GIF with agg (https://github.com/asciinema/agg).
#    Download the agg binary for your platform from its releases page, then:
agg --font-family "Cascadia Code,Consolas" --font-size 16 --fps-cap 10 --theme dracula \
    assets/demo/fuse-check-demo.cast assets/demo/fuse-check-demo.gif
```

The demo runs at whatever verification grade the environment supports: oracle grade
(speculative typecheck, no build) when a build capture is available for the fixture,
otherwise build grade. Either way the response text is the tool's real output.
