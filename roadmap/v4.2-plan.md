## R1. CI split for `RequiresSdk` tests

The review added `[Trait("Category", "RequiresSdk")]` and `SkipException.ForSkip`, but CI
still runs the full suite on every agent. SDK-heavy tests can pass vacuously or flake when the
runner lacks a working toolchain.

- Work: add a `ci-sdk.yml` job (or a filtered leg on `ci.yml`) that runs
  `dotnet test Fuse.slnx -c Release --filter "Category=RequiresSdk"` on win-x64 and linux-x64
  with the .NET 10 SDK and bundled build-capture worker; keep the default PR job on
  `Category!=RequiresSdk` for fast feedback. Document the split in `contributing.mdx`.
- Tests: the filtered command discovers the tagged tests; a PR that removes the trait fails the
  SDK job.
- Docs: `site/content/docs/project/contributing.mdx` (two CI legs and what each proves).
- Gate: both jobs green on a PR that touches a `RequiresSdk` test.

## R2. Finish `briefing.md` drift cleanup

`ext/vscode/`, and protocol mirrors removed in D15.

- Tests: ranking gate unchanged (`fuse eval ranking` on corpus v2); delete diagnostic-only

- Docs: `internals/scoping-internals.mdx` if it mentions flat FTS; `CHANGELOG.md` entry names

- Gate: `ranking.json` default configuration unchanged within CI on corpus v2.

## R4. Hook RPC verbose mode

Ambient verification hooks call `fuse host` over the pipe; `FuseHostClient` swallows RPC
failures so editors stay non-blocking.

- Work: add `FUSE_HOOK_VERBOSE=1` (or `--verbose` on the hook entry path if one exists) that
  logs RPC failures to stderr with method name and error code. Default remains silent.
- Tests: unit test on the client wrapper asserts stderr output when verbose and mocked failure.
- Docs: `internals/host-rpc.mdx` and `reference/environment-remediation.mdx` if hooks are
  documented there.
- Gate: default hook behavior unchanged; verbose mode covered by test.

## R5. Narrow `fuse update` process termination

`ToolUpdateLauncher` currently kills broad `fuse` peer processes, which is risky on shared CI
boxes and multi-repo workflows.

- Work: terminate only the updating tool's own child lineage or processes holding the same
  install mutex; add `--force-kill-peers` for the old behavior with a changelog warning.
- Tests: unit test with mocked process list asserts the narrow kill set; integration test
  optional.
- Docs: `site/content/docs/start/install.mdx` (staying up to date section).
- Gate: default update does not kill unrelated `fuse mcp serve` on another repo root (test with
  two temp roots if feasible, else unit-level).

## R7. Host IPC hardening
