Fuse v4 is out. It is a Model Context Protocol server that connects your AI coding agent
(Claude Code, Cursor, GitHub Copilot) to the .NET compiler.

Fuse typechecks a proposed edit before the agent writes it (`fuse_check`), computes the
blast radius of a change (`fuse_impact`), stages compiler-executed refactors as diffs that
either compile or are not handed over (`fuse_refactor`), and resolves how the code is
actually wired (which service implements an interface, which handler runs a request, which
action a route hits) from a Roslyn graph rather than file names and grep. Every answer
carries a grade so the agent knows what it is trusting, and Fuse abstains when it cannot
answer at compiler grade.

## The five numbers

Each is recorded under `tests/benchmarks/results` and reproduced with `fuse eval <suite>`.
The model-driven and opt-in numbers carry their scope in the same breath.

1. **It does not lie about an edit.** Over 1,000 compiler-labeled edits (500 breaking, 500
   neutral, generated by Roslyn rewriters and labeled by the compiler) plus 8 curated cases,
   `fuse_check` had zero false green and zero false red (`checkgate.json`).
2. **Agents finish more tasks with fewer silent failures.** On an agent loop comparison
   over the pinned benchmark corpus (driver `claude-sonnet-4-6`, 234 scored rollouts,
   confidence intervals, below the full arena size target), the Fuse arm reached a true
   pass@1 of 89 percent (95 percent CI 82 to 95)
   against native tools at 82 percent (95 percent CI 74 to 90), scored by a gold-test oracle
   rather than the agent claiming green, with false-done 8 versus 9 (`loop.json`).
3. **Truth without a build.** With the resident workspace enabled (opt-in `FUSE_RESIDENT=1`,
   environment-dependent), the delta-check answered at a P50 of 31.2 ms and a P95 of 42.4 ms
   at NodaTime scale (`resident-latency.json`).
4. **It reads .NET wiring exactly.** On the wiring fixture, the extracted semantic graph
   matches the hand-built edge ground truth exactly: 24 of 24 edges, recall and precision 1.0
   (`semantics.json`).
5. **A PR's context in about a thousand tokens.** Over 69 real merged pull requests,
   `fuse review` keeps 100 percent of the changed files at 93.4 percent precision in a median
   1,026 returned tokens; a grep baseline reaches 67 percent recall at 8 percent precision
   (`review.json`).

How the first two were measured, including the gate Fuse missed, is the subject of the
companion post: [1,000 compiler-labeled edits](/blog/compiler-labeled-edits).

## Install and connect

Two commands. Install the .NET global tool, then register it with your agent:

```bash
dotnet tool install -g Fuse
fuse mcp install --rules
```

The second command writes MCP config for Claude Code, Cursor, and GitHub Copilot in the
current project, and `--rules` adds a short instruction so the agent reaches for the `fuse_*`
tools instead of grepping blindly. The read tools build the index on first use.

## What is honest about the limits

- **Open-ended localization is the weakest mode.** From a task title alone, with no git base,
  `fuse localize` recalls 37.7 percent of the changed files (95 percent CI 30 to 46). Rather
  than return a low-precision candidate list on a no-signal title, Fuse refuses and hands back a
  navigation map. When a git base exists, the routed mode is change review, which is where the
  recall and precision are.
- **The deep graph is .NET-only.** Other languages are supported at the syntax tier
  (token-efficient context and search); the typed, compiler-backed graph is .NET, through
  Roslyn.
- **The millisecond delta-check is opt-in.** The resident workspace ships behind
  `FUSE_RESIDENT=1` and its latency is environment-dependent. The default verify path runs at
  oracle grade when a build capture is available and build grade otherwise, and names its
  grade either way.

The full method, the corpus, and the modes where Fuse loses are on the
[benchmarks page](/docs/project/benchmarks); the reproduction steps are on the
[reproduction page](/docs/project/reproduce). Every number is sourced, and the harness is in
the repository so you can run it yourself.
