Fuse v4 is out. It is a Model Context Protocol server that connects your AI coding agent
(Claude Code, Cursor, GitHub Copilot) to the .NET compiler.

Fuse typechecks a proposed edit before the agent writes it (`fuse_check`), computes the
typed impact of a change (`fuse_impact`), and stages compiler-executed refactors as
verify-gated diffs (`fuse_refactor`). It also resolves service, request, route, and
configuration wiring from a Roslyn graph. Compiler answers carry an evidence grade, and
Fuse abstains when no available compiler path can support a verdict.

## Five scoped observations

Each is recorded under `tests/benchmarks/results` and reproduced with `fuse eval <suite>`.
These results describe the named fixtures, corpora, and configurations.

1. **Edit classification on OrderingApp.** Across 1,000 compiler-labeled edits (500
   breaking and 500 neutral) plus 8 curated cases, `fuse_check` recorded zero false-green
   and zero false-red verdicts. This result does not cover every .NET edit
   (`checkgate.json`).
2. **Agent-loop result on a reduced-scope arena.** With `claude-sonnet-4-6` over 234 scored
   rollouts, the Fuse arm recorded true pass@1 of 89 percent (95 percent CI 82 to 95) and
   the native arm recorded 82 percent (95 percent CI 74 to 90). False-done counts were 8
   and 9. The confidence intervals overlap, and the arena was below its full size target
   (`loop.json`).
3. **Opt-in resident check latency.** With `FUSE_RESIDENT=1`, a dedicated NodaTime run
   measured delta-check latency at P50 31.2 ms and P95 42.4 ms over 25 iterations. The
   resident workspace is opt-in, and latency depends on the machine
   (`resident-latency.json`).
4. **Wiring agreement on a curated fixture.** The extracted graph matched all 24
   hand-built OrderingApp wiring edges with no false positives (`semantics.json`).
5. **Git-seeded review context.** Across 69 merged pull requests, `fuse review` retained
   every git-changed file at 93.4 percent precision in a median 1,026 returned tokens.
   Changed files were required seeds, so this is a precision result rather than evidence
   that review discovered those files (`review.json`).

How the first two were measured, including the gate that the run missed, is the subject of the
companion post: [1,000 compiler-labeled edits](/blog/compiler-labeled-edits).

## Install and connect

Two commands. Install the .NET global tool, then register it with your agent:

```bash
dotnet tool install -g Fuse
fuse mcp install --rules
```

The second command writes MCP config for Claude Code, Cursor, and GitHub Copilot in the
current project. The `--rules` option adds a short instruction describing when to call the
`fuse_*` tools. The read tools build the index on first use.

## Limits

- **Open-ended localization is the weakest mode.** From a task title alone, with no git base,
  `fuse localize` recalls 37.7 percent of the changed files (95 percent CI 30 to 46). Rather
  than return a low-precision candidate list on a no-signal title, Fuse refuses and returns a
  workspace map. When a git base exists, Fuse routes the task to change review.
- **The deep graph is .NET-only.** Other languages are supported at the syntax tier
  (token-efficient context and search); the typed, compiler-backed graph is .NET, through
  Roslyn.
- **The millisecond delta-check is opt-in.** The resident workspace ships behind
  `FUSE_RESIDENT=1` and its latency is environment-dependent. The default verify path runs at
  oracle grade when a build capture is available and build grade otherwise, and names its
  grade either way.

The full method, corpus, and weaker modes are on the
[benchmarks page](/docs/project/benchmarks); the reproduction steps are on the
[reproduction page](/docs/project/reproduce).

Install Fuse with the two commands above, confirm activation with `fuse_workspace
action=status`, then follow the [quickstart](/docs/start/quickstart) for the first checked
edit and wiring query.
