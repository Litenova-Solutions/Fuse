title: Reduction levels

---

**Reduction** is how Fuse shrinks content without changing what it means. A single dial,
the **reduction level**, drives the C# structural reduction: it is one enum value that
selects how far Fuse goes, from no structural change to signature-only skeletons. Pick
the smallest level that still answers your task.

The level is set with `--level <value>` on the CLI and the `level` parameter on the MCP
tools. The default is `none`.

## The Five Levels

| Level | What it does to C# | Use it when |
|-------|--------------------|-------------|
| `none` | No C# structural reduction. Whitespace trim and condense and the orthogonal flags below still apply. | Every fusion, a safe baseline |
| `standard` | Removes C# comments, `using` directives, namespace wrappers, and `#region` directives. | Trim noise but keep structure |
| `aggressive` | The `standard` removals plus aggressive whitespace and syntax compression. | Maximum reduction with bodies intact |
| `skeleton` | Emits structural skeletons (type and member signatures), bodies removed. | An architecture pass on an unfamiliar solution |
| `publicApi` | Like `skeleton`, but keeps only public and protected members. | The contract a project presents to consumers |

The levels are cumulative in effect: `aggressive` includes everything `standard` does,
and `publicApi` is `skeleton` narrowed to the public surface.

### none

The default does lossless cleanup only: it normalizes whitespace, condenses blank lines,
and minifies XML-family and Razor files. It applies no C# structural reduction. This runs
on every fusion regardless of level.

### standard

`--level standard` removes four categories of C# structure together:

- Comments: line and block comments, preserving string contents.
- Using directives and aliases.
- Namespace declarations and their indentation.
- `#region` markers and preprocessor directives.

These four were once separate flags; the level applies all of them at once.

### aggressive

`--level aggressive` runs the `standard` removals and then a second tier: it strips noise
attributes such as `DebuggerDisplay` and `MethodImpl`, removes the `this.` prefix,
rewrites auto-properties to a compact form, and collapses whitespace around delimiters.
These transforms are **regex-based**, not Roslyn-parsed. They preserve string-literal
contents and keep 99 to 100 percent of public types and methods on the benchmark corpus,
but the output is no longer guaranteed to compile and can rewrite syntax in ways a
Roslyn parse would reject. Use it when the reader is a model that needs the logic, not
a compiler. On the benchmark corpus it removes 21 to 46 percent of tokens at full type
and method fidelity, which makes it the dependable cut for bodies intact.

### skeleton

`--level skeleton` drops method bodies entirely and emits class, interface, and method
signatures only. Unlike `aggressive`, skeleton extraction uses **Roslyn syntax analysis**,
so signatures match the source parse tree rather than a line-based regex guess. That
makes skeleton the faithful architecture map: on the benchmark corpus it removes 39 to
56 percent of tokens and keeps every method signature, including on Newtonsoft.Json where
the retired regex skeleton kept only 4 percent. The Roslyn skeleton is larger than the
old regex skeleton because the regex cut was partly a fidelity failure, not real saving.
See [Roslyn structural analysis vs regex](/docs/concepts/precision-tier) for the
measured comparison.

### publicApi

`--level publicApi` is the skeleton narrowed to the contract a project exposes: it keeps
only public and protected member signatures and drops the rest. Reach for it when you want
the API surface alone.

## The Orthogonal Flags

The level is the single dial for C# structural reduction. A set of other options are
separate and orthogonal: they are not folded into the level and apply at any level. They
include best-effort secret redaction (`--no-redact`, `--redact-report`; regex and entropy heuristics), generated-code collapse
(`--collapse-generated`), the structural annotations (`--semantic-markers`), the pattern
summary (`--pattern-summary`), the route map (`--route-map`), the project graph
(`--project-graph`), XML and HTML/Razor minification (`--minify-xml-files`,
`--minify-html-and-razor`), and the per-line content handling (trim and condense).

Generated-code collapse is separate from the level. Combine it when you need both:

```bash
fuse dotnet --directory ./src --level aggressive --collapse-generated
```

## Reduction Is Not Deletion

The `none`, `standard`, and `aggressive` levels are designed to keep the public API. An
independent Roslyn oracle confirms 99 to 100 percent of public types and methods survive,
and `fuse verify` reports the preserved surface for any run. The `skeleton` and
`publicApi` levels make no claim of body fidelity, by design: they keep signatures and
drop bodies.

## Survey Before You Reduce

When you only need a map, the table-of-contents survey is cheaper still:

```bash
fuse dotnet --directory ./src --toc
```

`--toc` emits a directory tree with a per-file symbol outline and token cost instead
of file bodies. See [Survey a codebase cheaply](/docs/scenarios/survey-cheaply).
