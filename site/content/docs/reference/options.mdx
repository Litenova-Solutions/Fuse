Fuse commands take a workspace path argument and a small set of per-command flags. This page lists the flags for each command. The commands themselves, and when to reach for each, are in [Commands](/docs/reference/commands); the MCP tool parameters are in [Tools](/docs/reference/mcp-tools).

This page is for engineers and maintainers who need exact flag behavior and defaults. Types use these conventions: `bool` is a switch, `int` is a whole number, `string` is a single value, and `string[]` is a repeatable or space-separated list.

## Purpose and scope

The read commands build the persistent index on first use, so most commands take only the workspace path plus the flags that shape their query or output. Token budgets are honored by the plan, and secret redaction runs before any source reaches the payload (it is always on; there is no flag to disable it).

## fuse index

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory to index. |
| `--force` | bool | `false` | Force a full re-index instead of an incremental update. |

## fuse map

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory to map. |
| `--detail` | string | `all` | What to include: `symbols`, `routes`, or `all`. |
| `--max-rows` | int | `200` | Maximum rows per section. |

## fuse localize

Supply `--task` for an open-ended query, or one of the named flags to seed candidates from a specific node.

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory. |
| `--task` | string | None | Free-text task or query to localize. |
| `--route` | string | None | A route to resolve, for example `POST /api/orders/{id}`. |
| `--symbol` | string | None | A symbol to focus on. |
| `--service` | string | None | A service to resolve to its implementation. |
| `--request` | string | None | A request or command to resolve to its handler. |
| `--config` | string | None | A config section to resolve to its options type. |
| `--changed-since` | string | None | A git base ref whose changed files seed the candidates. |
| `--max-candidates` | int | `50` | Maximum candidates to return. |

## fuse resolve

Supply exactly one of the five.

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory. |
| `--service` | string | None | Resolve a service to its implementation. |
| `--request` | string | None | Resolve a request or command to its handler. |
| `--route` | string | None | Resolve a route to its handler. |
| `--config` | string | None | Resolve a config section to its options type. |
| `--symbol` | string | None | Resolve a symbol name to its declaration. |

## fuse context

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory. |
| `--seed` | string[] | None | A named seed (symbol, service, request, or config section). Repeatable. |
| `--route` | string[] | None | A route seed, for example `POST /api/orders/{id}`. Repeatable. |
| `--depth` | int | `2` | Graph expansion depth. |
| `--max-tokens` | int | None | Token budget; must-keep seeds are always included. |
| `--format` | string | `xml` | Output format: `xml`, `markdown`, or `json`. |
| `--plan-only` | bool | `false` | Print the plan without rendering source bodies. |

## fuse review

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory. |
| `--changed-since` | string | `HEAD` | Git base ref to diff against (branch, commit, or `HEAD~N`). |
| `--max-tokens` | int | None | Token budget; changed files are always kept. |
| `--include-tests` | bool | `true` | Include related test files in the blast radius. |
| `--format` | string | `xml` | Output format: `xml`, `markdown`, or `json`. |
| `--plan-only` | bool | `false` | Print the plan without rendering source bodies. |

## fuse find

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `query` | string | required | The name, path fragment, or text to find. |
| `--path` | string | `.` | Workspace directory. |
| `--kind` | string | `all` | Restrict to one kind: `symbol`, `path`, `text`, or `all`. |

## fuse diagnostics

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `path` | string | `.` | Workspace directory whose index state to report. |

## fuse reduce

Compact a known set of files or piped content without walking a directory. Pass `--files` or `--stdin`, not both.

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--files` | string[] | None | File paths to reduce (space-separated). Mutually exclusive with `--stdin`. |
| `--stdin` | bool | `false` | Read content to reduce from standard input. Mutually exclusive with `--files`. |
| `--ext` | string | `.cs` | Extension that selects the reducer for `--stdin` content (for example `.cs`, `.ts`, `.py`). |
| `--level` | string | `standard` | Reduction level: `none`, `standard`, `aggressive`, `skeleton`, or `publicApi`. |
| `--directory` | string | `.` | Base directory for resolving relative `--files` paths. |
| `--max-tokens` | int | `0` | Maximum tokens the reduced output may use (`0` means no limit). |

The reduction levels and what each does to C# are in [Reduction levels](/docs/concepts/reduction-levels).

## Reduction levels

A single `--level` value drives C# structural reduction wherever it applies (`fuse reduce`, and the render tier `fuse context` picks per file).

| Level | What it does to C# |
|-------|--------------------|
| `none` | No structural reduction; whitespace trim only. |
| `standard` | Removes comments, `using` directives, namespace wrappers, and `#region` directives. |
| `aggressive` | The `standard` removals plus whitespace and syntax compression. |
| `skeleton` | Emits type and member signatures, bodies removed. |
| `publicApi` | Like `skeleton`, but keeps only public and protected members. |

## Other commands

`fuse init`, `fuse mcp install`, `fuse mcp serve`, and `fuse host` take no query flags from this set. See [Commands](/docs/reference/commands) for their behavior.
