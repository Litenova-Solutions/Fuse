---
title: Ask one question
description: "Get the answer to a single question about a .NET codebase in one call: localize an open-ended question, or resolve a named piece of wiring."
---

**Goal:** answer one question about a codebase, such as "where is the discount applied?"
or "what handles this request?", without reading file after file.

Fuse answers this against the warm index in one call. Which tool depends on whether the
question names a concrete piece of wiring or is open-ended.

## A Named Question: Resolve

When the question names a service, request, route, or config section, `fuse_resolve`
answers it deterministically by walking the graph to the real target:

```text
fuse_resolve(path="C:/Projects/MyApp/src", request="CreateOrderCommand")
fuse_resolve(path="C:/Projects/MyApp/src", route="POST /api/orders/{id}")
fuse_resolve(path="C:/Projects/MyApp/src", service="IOrderService")
```

Each returns the resolved target (the handler, the action, the concrete implementation)
with provenance and no bodies. This is the answer text search cannot give: it finds the
type that actually runs, not the one a name matches.

## An Open-Ended Question: Localize

When the question is a topic rather than a name, `fuse_localize` ranks the files and
symbols most likely to hold the answer, with reasons and token costs and no bodies:

```text
fuse_localize(
  path="C:/Projects/MyApp/src",
  task="where is the discount applied at checkout?"
)
```

One scoped localize call is the measured shape of one-call context: in the
[Layer 4 scenario](/docs/project/benchmarks) it acquired a task's context in about 37,000
tokens (49 percent recall of the changed files) against roughly 425,000 tokens for a
generic-packer dump at the same single call.

## When the Question Has No Signal

If the question names no code (a merge note, a dependency bump, or an empty query with no
route, symbol, or base), `fuse_localize` declines rather than guessing: it returns a
low-signal verdict and suggests a next input (a git base, a route, or a symbol) instead of
full-text candidates. Declining beats returning files a vague title cannot actually point
at. Provide one of the structured inputs (a `route`, `symbol`, `service`, `request`,
`config`, or `changedSince` base) and call again.

## Then Fetch the Bodies

Both tools return locations, not source. When you want the code, feed the result to
`fuse_context`:

```text
fuse_context(path="C:/Projects/MyApp/src", requests=["CreateOrderCommand"], maxTokens=15000)
```

## When to Use It

Use `fuse_resolve` when the question names wiring you can quote (an interface, a request,
a route, a config section) and `fuse_localize` when it is a topic. The full progressive
flow for a larger task is in [Context for an agent](/docs/scenarios/context-for-an-agent).
