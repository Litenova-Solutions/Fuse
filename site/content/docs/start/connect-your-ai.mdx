title: Connect your coding agent

Connect Fuse to Claude Code, Cursor, or GitHub Copilot from the current .NET project:

```bash
fuse mcp install --rules
```

Model Context Protocol (MCP) is the connection format that lets a coding agent launch
Fuse and call its local tools. You do not keep `fuse mcp serve` running in a terminal.
The agent starts it when needed.

Reload the agent, then ask it to check the connection:

```text
Call fuse_workspace with action=status for this repository. Return the workspace path,
analysis mode, and freshness exactly as Fuse reports them.
```

A response containing the current repository path confirms activation. A list of
available tools does not.

## Choose the Registration Scope

The opening command registers all three supported clients for the current project. To
register Fuse for every project on this machine:

```bash
fuse mcp install --rules --scope user
```

To register one client:

```bash
fuse mcp install --client cursor --rules
fuse mcp install --client claude --rules
fuse mcp install --client copilot --rules
```

The available flags are:

- `--client claude|cursor|copilot|all`, default `all`
- `--scope project|user`, default `project`
- `--rules`, which writes the recommended agent instructions
- `--command`, which changes the executable the client launches

## What Project Setup Writes

- Claude Code: `.mcp.json`, plus a Fuse-managed block in `CLAUDE.md` when `--rules` is set.
- Cursor: `.cursor/mcp.json`, plus `.cursor/rules/fuse.mdc` when `--rules` is set.
- GitHub Copilot: `.vscode/mcp.json`, plus a Fuse-managed block in
  `.github/copilot-instructions.md` when `--rules` is set.

Existing JSON and instruction files are merged rather than replaced. Re-running the
command updates Fuse-managed entries in place. Project files can be committed so the team
receives the same setup.

At user scope, Claude registration uses `claude mcp add` and writes its rule to
`~/.claude/CLAUDE.md`. Cursor writes `~/.cursor/mcp.json`. Copilot writes the VS Code user
`mcp.json` under:

- Windows: `%APPDATA%\Code\User`
- macOS: `~/Library/Application Support/Code/User`
- Linux: `~/.config/Code/User`

Cursor and Copilot do not have a user-global rule file, so `--rules --scope user` skips
those two rules with a note.

The optional `--with-hooks` flag also merges two Claude Code hooks into
`.claude/settings.json`: a post-edit `fuse check --delta --fast` hook and a stop-time
`fuse gate` hook.

## Get the First Compiler-Backed Result

After the status call succeeds, ask the agent:

```text
Before editing OrderService.cs, call fuse_check with the proposed file content. Show the
compiler result before writing the change.
```

Fuse labels the compiler path used for each check. This label is the verification grade:
oracle grade uses a build-captured compilation, while build grade runs a scoped real
`dotnet build`. If neither path can answer, Fuse abstains and states what is missing.

The first call builds the local `.fuse/fuse.db` index. A large repository can take from a
few seconds to tens of seconds on that cold call. Fuse reports `partial` or `syntax`
analysis when MSBuild cannot load every project.

## Local Process and Network Behavior

The client launches `fuse mcp serve` as a child process and communicates over standard
input and output. Fuse does not listen on a TCP port. Source files and the derived index
remain on your machine, and Fuse does not send source to a hosted Fuse service.

Your coding agent still applies its own data policy to prompts and tool results. Fuse can
contact NuGet for a daily update check; set `FUSE_UPDATE_CHECK=0` in the server environment
to disable it. A compiler fallback can run the repository's normal `dotnet build`,
including any package-feed access required by that build.

## Manual Registration

Use these files if you prefer to edit the client configuration yourself.

Claude Code, `.mcp.json`:

```json
{
  "mcpServers": {
    "fuse": {
      "type": "stdio",
      "command": "fuse",
      "args": ["mcp", "serve"]
    }
  }
}
```

You can also use the Claude CLI:

```bash
claude mcp add fuse --scope project -- fuse mcp serve
```

Use `--scope user` to make it available in every project.

Cursor, `.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "fuse": {
      "command": "fuse",
      "args": ["mcp", "serve"]
    }
  }
}
```

GitHub Copilot in VS Code, `.vscode/mcp.json`:

```json
{
  "servers": {
    "fuse": {
      "type": "stdio",
      "command": "fuse",
      "args": ["mcp", "serve"]
    }
  }
}
```

## Roll Back the Setup

There is no `fuse mcp uninstall` command. Remove only the `fuse` entry from each client
configuration file listed above, preserving other entries. For Claude user scope, run:

```bash
claude mcp remove fuse --scope user
```

Delete `.cursor/rules/fuse.mdc`. In `CLAUDE.md`,
`.github/copilot-instructions.md`, or `~/.claude/CLAUDE.md`, delete the block from
`<!-- fuse:begin ... -->` through `<!-- fuse:end -->`. If you installed hooks, delete the
two Fuse entries from `.claude/settings.json` under `hooks`.

Stop or reload the coding agent, then delete `.fuse/` at the repository root to remove
the derived index. Deleting `.fuse/` does not change source files. A later Fuse read
rebuilds it.
