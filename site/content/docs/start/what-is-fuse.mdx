title: What is Fuse
description: Fuse collapses an AI agent's explore phase into one scoped call, finding the code a task needs and handing it over reduced, with the public API intact.
---

An AI coding agent spends the start of every task exploring: listing directories,
searching, opening file after file to find the few that matter. Fuse collapses that
explore phase into one scoped call: it finds the code a task needs and hands it over
already reduced, so the agent starts work instead of hunting.

The collapse is measured. Over 24 real merged pull requests, one scoped call acquires a
change's context in about 40,000 tokens at 51 percent recall, against a blind agent that
must read each needed file at least once (a structural lower bound of about six reads)
and a generic packer that dumps the whole repository at roughly 512,000 tokens. Read the
token number with its recall; the change-scoping mode reaches 88 percent recall when a
git base is available. See [The explore phase](/docs/concepts/the-explore-phase) and the
[Layer 4 benchmark](/docs/project/benchmarks).

Hand it to an agent through the MCP server: give `fuse_ask` a task and a token budget and
it picks the scoping strategy and packs to fit.

```text
fuse_ask(
  path="C:/Projects/MyApp/src",
  task="where is the discount applied at checkout?",
  tokenBudget=20000
)
```

It also runs as a CLI for humans and CI:

```bash
# install once
dotnet tool install -g Fuse

# fuse a project, keeping the full public API
fuse dotnet --directory ./src --level aggressive
```

```text
Fused 511 files
Estimated tokens: 366,121 (-21.5%)
cache: 0 hit / 511 miss
Output: AutoMapper_2026-06-20_366k.txt
```

That run took 511 C# files down by 21.5 percent and kept 99 percent of public types
and 100 percent of public methods. The cut is not deletion: the surface an agent
reads the code for is still there.

## What It Does, in One Line

Fuse is a retrieval engine for .NET coding agents: it finds the code a task needs and
hands it over scoped and reduced in one call, instead of letting the agent open thousands
of files to find its way around.

A single request returns a **fusion**: one scoped, reduced view of a codebase assembled
for that request, returned as an MCP tool response to an agent or written to a file from
the CLI.

## Why That Matters

On a solution with hundreds or thousands of C# files, the explore phase is the expensive
part of the task: every file the agent opens to orient itself burns context window, each
open is a separate round-trip with latency, and reading files in isolation loses the
dependency structure. Fuse front-loads that work into one deterministic step and hands
the agent scoped, reduced context that fits the window. For the full mechanism, see
[The explore phase](/docs/concepts/the-explore-phase).

## Reduction without Losing the Surface

Across a commit-pinned corpus of real .NET libraries, the default and `--level aggressive`
reductions cut 7 to 40 percent of tokens while keeping 99 to 100 percent of public
types and methods. Skeleton mode, which keeps signatures and drops bodies, cuts 66
to 93 percent for an architecture-level survey.

## Two Ways to Run It

Fuse ships from one package in two forms:

- The **`fuse` global tool** writes a fusion to a file from your terminal. Start at
  the [Quickstart](/docs/start/quickstart).
- The **`fuse mcp serve` MCP server** exposes eight tools to an AI client such as Claude
  Code, Cursor, or GitHub Copilot, so the agent requests scoped context directly.
  See [Connect to your AI](/docs/start/connect-your-ai).

## What It Is Not

Fuse is not a generic file packer that concatenates text, and it is not an embedding
index that returns fuzzy chunks. It is also not a one-shot dump: it scopes to the files a
task needs and refines across calls in a session. It understands C# structure, preserves
the public API, and produces the same output for the same input. The
[Why Fuse](/docs/start/why-fuse) page covers the landscape and the trade-offs in full.
